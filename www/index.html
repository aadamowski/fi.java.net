Fast Infoset Project

Welcome to the Fast Infoset Project. This is a project the Java community on Web Services and XML at java.net.

Fast Infoset specifies a binary encoding for the XML Information Set.  An XML infoset (such as a DOM tree, StAX events or SAX events in programmatic representations) may be serialized to an XML 1.x document or, as specified by Fast Infoset, may be serialized to a fast infoset document. Fast infoset documents are generally smaller in size and faster to parse and serialize than equivalent XML documents.

Analogies taken too far can lead to all sorts of misconceptions but one can think of Fast Infoset as 'gzip' for XML:

  • Fast Infoset optimizes the properties of size and performance, gzip optimizes only the property of size; and
  • Fast infoset documents are generally larger than gzipped XML documents, however for small XML infosets this may not be always be the case.

Goals

The goals of the Fast Infoset Project are to:

  1. Develop high performance Java-based implementations of Fast Infoset serializers and parsers. Such implementations will be mostly based on standard XML APIs such as SAX or StAX. Extensions to such APIs will be required to support the encoding of binary data;
  2. Develop useful functionality in support of the Fast Infoset implementations;
  3. Integrate Fast Infoset serializers and parsers into Java-based distributed communication platforms, such as Java-based Web services; and
  4. Last but not least, foster a community of people interested in developing and/or using Fast Infoset.

Fast Infoset specification and standardization

The Fast Infoset specification, ITU-T Rec. X.891 | ISO/IEC 24824-1 (Fast Infoset), is being standardized at the ITU-T and ISO.

It is currently at the stage of First Committee Draft in ISO and is under ballot, which finishes on the 1st of February 2005. The specification will then go to Last Call in the ITU-T at the next Study Group 17 meeting (February or March/April 2005).

Unfortunately the specification is not available publically. It is possible to get access the specification from the:
  • ITU-T if you or your company is an ITU-T sector member, or a Study Group 17 associate member; or from
  • Your ISO National Body.

Relevant to the Fast Infoset specification are two further specifications under development:

  1. Fast Web Services specifies alternative binary encodings for SOAP messages without the need to modify WSDL service descriptions (the SOAP binding and ports do not need to be augmented with new bindings and ports). Fast Infoset may be used in conjunction with an optimized binary encoding of SOAP 1.2 messages (referred to as ASN.1 SOAP messages) or for the serialization of SOAP message infosets (SOAP 1.1 or SOAP 1.2) as fast infoset documents.
  2. Security in Fast Infoset and Fast Web Services is a new proposed work item that aims to specify how standard XML security solutions, such as XML Signature, XML Encryption and SOAP Message Security, may be applied to Fast Infoset and Fast Web Services.

Features

There are a number of features of Fast Infoset some of which are highlighted in this article. A new article is required to fully explain all the features of Fast Infoset.

Two important features of Fast Infoset are:

  1. Support for vocabularies defined externally, internally at the head or dynamically generated in the content of a fast infoset document. A vocabulary is a set tables consisting of strings and qualified names associated with an XML document, such strings may correspond to element and attribute names or namespace prefixes and namespace names. Using an external vocabulary, that is referenced by a URI, allows for smaller documents since the vocabulary of the document need not be encoded. A vocabulary at the head of a document allows for serializers to assign the smallest indexes to the most commonly repeated strings. Vocabulary dynamically in the content of the document allows for the support of streaming applications and extensibility of an XML infoset. All three forms of vocabulary may be used together.
  2. Support for binary encoding algorithms. Fast Infoset specifies built-in encoding algorithms such as for the encoding of integers and floating pointer numbers in binary form rather than as characters.  Encoding algorithms specific to a vocabulary (external or at the head) may also be specified using URIs. This is useful for domain specific areas such as 2D or 3D graphics that specify algorithms for the encoding of co-ordinate data in binary form rather than as characters.

Current state of affairs

Currently information is a bit thin on the ground. In due course this project will be updated with more detailed information on what is present and what needs to be done. However, there should be enough information available to compile and play with the source.

The source contains partial implementations of the Fast Infoset specification for the SAX API and the StAX API. The SAX API is more complete. Both support external vocabularies and dynamic generation. Support for encoding algorithms is required, which may require additional interfaces to SAX and StAX so that data may be accessed in non-character form.

There are two routes to building the source:

  • Install NetBeans 4.0 and use either Ant (version 1.6.x) or the IDE to build the project; or
  • Use the alternative ant build script, build-without-nb.xml, for the case where NetBeans 4.0 is not installed.
A number of UNIX based scripts are present to convert XML documents to fast infoset documents and vice versa. In addition other scripts will test that it is possible to round trip an XML infoset to and from fast infoset documents. For more information see the README.txt file.
